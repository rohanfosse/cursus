# Task: Service lumenSnapshot — parse URL + fetch GitHub + validation

## Description

Creer `server/services/lumenSnapshot.js` qui transforme une URL GitHub publique en un snapshot JSON complet (arborescence + contenu base64), avec validation stricte des limites et gestion explicite des cas d'erreur.

## Acceptance Criteria

- [ ] Fonction `parseGitHubUrl(url)` : extract `{ owner, repo }` depuis `https://github.com/owner/repo` ou `https://github.com/owner/repo.git`. Throw `InvalidUrlError` si format invalide.
- [ ] Fonction `fetchRepoMetadata(owner, repo)` : appelle `GET https://api.github.com/repos/:owner/:repo` avec User-Agent header. Retourne `{ default_branch, size }`. Throw `RepoNotFoundError` sur 404, `RateLimitError` sur 403 avec reset time, `FetchError` sinon.
- [ ] Fonction `fetchRepoTree(owner, repo, ref)` : appelle `GET /repos/:owner/:repo/git/trees/:ref?recursive=1`. Retourne `[{ path, type, sha, size }]` filtre sur `type === 'blob'`.
- [ ] Fonction `fetchFileContent(owner, repo, path)` : appelle `GET /repos/:owner/:repo/contents/:path`. Retourne `{ content_base64, size, sha }`.
- [ ] Fonction `buildSnapshot(url)` orchestre tout : valide les limites dures au fur et a mesure, throw `SnapshotLimitError` avec compteurs precis si depasses.
- [ ] Limites dures validees :
  - Max 200 fichiers
  - Max 5 242 880 octets (5 Mo) total apres decode base64
  - Max 524 288 octets (512 Ko) par fichier individuel
- [ ] Timeout global de 30s sur l'ensemble du fetch (via `AbortController`)
- [ ] Format de sortie du snapshot :
  ```json
  {
    "repo_url": "https://github.com/owner/repo",
    "default_branch": "main",
    "commit_sha": "abc123",
    "fetched_at": "2026-04-08T...",
    "files": [{ "path": "main.py", "size": 1234, "content_base64": "..." }],
    "total_size": 1801,
    "file_count": 2
  }
  ```
- [ ] Les erreurs exposent des codes stables : `INVALID_URL`, `REPO_NOT_FOUND`, `RATE_LIMIT`, `TIMEOUT`, `TOO_MANY_FILES`, `TOTAL_SIZE_EXCEEDED`, `FILE_SIZE_EXCEEDED`, `FETCH_ERROR`

## Technical Details

- Utiliser `fetch` natif Node >= 18
- Ajouter header `Accept: application/vnd.github+json` et `User-Agent: Cursus-Lumen-Snapshot`
- Pas de token GitHub (repos publics uniquement) — le rate limit de 60 req/h par IP est accepte
- SSRF protection : whitelist `api.github.com` uniquement, pas de follow redirects vers d'autres hosts
- Les fichiers binaires sont stockes en base64 tel quel (GitHub retourne deja en base64 via l'endpoint `contents`)
- Structure du service : exporter `{ buildSnapshot, SnapshotError, ErrorCodes }`
