# Task: Composants LumenProjectTree + LumenProjectFileViewer

## Description

Creer deux composants Vue autonomes pour afficher l'arborescence d'un projet snapshote (`LumenProjectTree`) et le contenu d'un fichier individuel avec coloration syntaxique (`LumenProjectFileViewer`).

## Acceptance Criteria

### LumenProjectTree.vue

- [ ] Props : `files: { path: string; size: number }[]`, `activePath: string | null`
- [ ] Emit : `select(path)` au clic sur un fichier
- [ ] Construit une structure hierarchique a partir de la liste plate de paths (split sur `/`)
- [ ] Dossiers collapsibles avec icone chevron, memorise l'etat expand/collapse par dossier
- [ ] Icones differenciees par extension : `.py`, `.js`, `.ts`, `.md`, `.json`, `.html`, `.css`, fallback fichier generique (lucide icons)
- [ ] Indentation visuelle par niveau de profondeur
- [ ] Fichier actif : highlight avec `background` et `border-left` accent
- [ ] Taille du fichier affichee a droite en `Kb` si > 1 Ko
- [ ] Navigation clavier : fleches haut/bas pour bouger, droite pour expand, gauche pour collapse, Enter pour select
- [ ] A11y : `role="tree"`, `aria-expanded` sur les dossiers, `aria-selected` sur le fichier actif

### LumenProjectFileViewer.vue

- [ ] Props : `course_id: number`, `path: string | null`
- [ ] Etats : loading, error, binary, text
- [ ] Au changement de `path` : appelle `store.fetchFileContent(course_id, path)` avec cache
- [ ] Detection binaire : si le contenu decode base64 contient des bytes non-UTF8 ou si l'extension est binaire (`.png`, `.jpg`, `.pdf`, `.zip`, `.ico`), affiche "Fichier binaire — telecharge le zip pour le recuperer"
- [ ] Detection du langage via `hljs.getLanguage(ext)` ou `hljs.highlightAuto` fallback
- [ ] Rendu via `highlight.js` en reutilisant la config de [markdown.ts](src/renderer/src/utils/markdown.ts)
- [ ] Affichage des numeros de ligne
- [ ] Container scrollable horizontalement pour les lignes longues
- [ ] Header avec : path complet, taille, bouton "Copier" (contenu texte)
- [ ] Empty state si `path === null` : "Clique sur un fichier dans l'arborescence"

## Technical Details

- Les deux composants sont purement presentationnels (pas de Pinia direct dans Tree, File fait l'appel store)
- Emplacement : `src/renderer/src/components/lumen/LumenProjectTree.vue` + `LumenProjectFileViewer.vue`
- Reutiliser `hljs` deja importe dans markdown.ts via une export intermediaire si besoin
- Cache du contenu decode dans le store (cf. tache 127)
- Les extensions binaires sont listees en constante en tete de fichier viewer
