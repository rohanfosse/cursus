# Task: Tests frontend (tree, viewer, integration reader)

## Description

Couverture de tests vitest pour les trois composants frontend (`LumenProjectTree`, `LumenProjectFileViewer`, `LumenProjectPanel`) et verification de l'integration dans `LumenReader`.

## Acceptance Criteria

### LumenProjectTree

- [ ] Rendu d'une arborescence a partir d'une liste de paths plate
- [ ] Clic sur un fichier emet `select(path)` avec le bon path
- [ ] Expand/collapse d'un dossier cache/affiche ses enfants
- [ ] Highlight du fichier actif via prop `activePath`
- [ ] Navigation clavier : fleches haut/bas changent la selection, droite expand, gauche collapse, Enter select
- [ ] Role ARIA `tree` et `aria-expanded` presents

### LumenProjectFileViewer

- [ ] Charge le contenu via mock du store au changement de prop `path`
- [ ] Affiche un loading state pendant le fetch
- [ ] Rend le contenu avec coloration (verifier la presence des classes hljs)
- [ ] Fichier binaire : affiche le message "Fichier binaire — telecharge le zip"
- [ ] Cache : deux fetches successifs sur le meme path n'appellent le store qu'une fois
- [ ] Bouton "Copier" : ecrit dans le presse-papier (mock `navigator.clipboard`)
- [ ] Empty state visible si `path === null`

### LumenProjectPanel

- [ ] Rend Tree + Viewer cote a cote
- [ ] Clic dans le tree met a jour `activePath` et charge le viewer
- [ ] Bouton download visible et construit la bonne URL
- [ ] Badge avec `file_count` et taille totale
- [ ] Lien "Voir sur GitHub" pointe vers `repo_url`

### Integration LumenReader

- [ ] Le panneau n'apparait pas si `course.repo_snapshot_meta` est null
- [ ] Le panneau apparait si meta defini
- [ ] Pas de regression sur le TOC, la progress bar, prev/next (tests existants)

## Technical Details

- Framework : vitest + @vue/test-utils
- Mock du store Pinia via `createTestingPinia`
- Fixture de snapshot avec 5-10 fichiers simulant un vrai projet Python
- Les tests de navigation clavier utilisent `trigger('keydown', { key: 'ArrowDown' })`
- Emplacement : `tests/frontend/components/lumen/*.test.ts`
