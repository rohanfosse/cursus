# Task: Routes REST snapshot + integration publish

## Description

Ajouter les routes REST pour le cycle complet du snapshot (creation, refresh, download, lecture de fichier) et integrer le snapshot dans le flow de publication existant.

## Acceptance Criteria

- [ ] `POST /api/lumen/:id/snapshot` â€” declenche `buildSnapshot(course.repo_url)` et stocke le resultat. Requiert ownership teacher du cours. Retourne `{ ok: true, file_count, total_size, commit_sha }` ou erreur structuree.
- [ ] `POST /api/lumen/:id/snapshot/refresh` â€” alias explicite de la precedente (pour clarte cote UI). Refetch le repo, compare les SHA, retourne `{ ok, changed: boolean }`.
- [ ] `GET /api/lumen/:id/snapshot/download` â€” streaming du zip via `lumenZip.streamZipFromSnapshot`. Header `Content-Disposition: attachment; filename="{slug}-exemple.zip"`.
- [ ] `GET /api/lumen/:id/snapshot/file?path=...` â€” retourne `{ path, content_base64, size }` d'un fichier individuel. Valide que le path appartient au snapshot (pas de traversal). 404 si inconnu.
- [ ] `POST /api/lumen/:id/publish` etendu : si `repo_url` est defini et pas de snapshot existant â†’ appelle `buildSnapshot` avant de marquer le cours publie. Si l'URL a change depuis le dernier snapshot â†’ refetch. Si echec â†’ retourne l'erreur sans publier.
- [ ] Les erreurs du service sont mappees en codes HTTP : `INVALID_URL` â†’ 400, `REPO_NOT_FOUND` â†’ 404, `RATE_LIMIT` â†’ 429, `TOO_MANY_FILES`/`TOTAL_SIZE_EXCEEDED`/`FILE_SIZE_EXCEEDED` â†’ 413, `TIMEOUT` â†’ 504, `FETCH_ERROR` â†’ 502.
- [ ] Toutes les routes sont soumises au middleware `requireAuth` + check ownership teacher pour les endpoints mutation.
- [ ] `GET /:id/snapshot/download` et `/file` accessibles aux etudiants de la promo du cours (reader mode).

## Technical Details

- Fichier : `server/routes/lumen.js` (etendre l'existant)

- Pour l'acces etudiant au download : verifier que l'etudiant appartient a `course.promo_id`
