# Task: IPC preload + actions store lumen.ts

## Description

Exposer les nouvelles routes backend au renderer via IPC et ajouter les actions correspondantes au store Pinia `lumen`.

## Acceptance Criteria

- [ ] `src/preload/index.ts` expose :
  - `updateLumenRepo(courseId, repoUrl)` → POST update du champ `repo_url` du cours (via l'endpoint `PUT /api/lumen/:id` existant, etendu)
  - `refreshLumenSnapshot(courseId)` → POST `/snapshot/refresh`
  - `getLumenFile(courseId, path)` → GET `/snapshot/file?path=...`
  - `downloadLumenSnapshotUrl(courseId)` → retourne l'URL complete avec token d'auth pour utiliser dans `<a href>` ou `window.open`
- [ ] Types IPC mis a jour dans `src/preload/index.d.ts`
- [ ] Store `src/renderer/src/stores/lumen.ts` etendu :
  - `setCourseRepo(courseId, url)` — update optimiste + appel backend
  - `refreshSnapshot(courseId)` — appelle IPC, update le state sur succes, gere les erreurs avec toast
  - `fetchFileContent(courseId, path)` — cache local par cours+path (Map) pour eviter refetch
- [ ] Type `LumenCourse` etendu avec `repo_url`, `repo_snapshot_meta` (subset leger : `{ commit_sha, default_branch, snapshot_at, file_count, total_size }` pour l'UI — pas le contenu complet)
- [ ] Le payload des endpoints `GET /lumen/courses` et `GET /lumen/:id` est adapte pour ne pas charger `repo_snapshot` complet par defaut (juste le meta) — une option `?include=snapshot` ou une route dediee pour l'arborescence

## Technical Details

- Pour l'arborescence affichee au reader, ajouter un endpoint `GET /api/lumen/:id/snapshot/tree` qui retourne uniquement la liste `[{ path, size }]` sans les contenus (pour eviter de downloader 5 Mo de base64 juste pour afficher l'arbre)
- Le contenu d'un fichier n'est fetch qu'au clic (lazy)
- Cache cote store : `Map<string, string>` avec cle `${courseId}:${path}`
