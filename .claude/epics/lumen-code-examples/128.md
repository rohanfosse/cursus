# Task: UI editor — meta row URL repo + badge + resync

## Description

Ajouter a la meta row de l'editeur Lumen un champ URL repo git, un badge d'etat du snapshot (apres publication), et un bouton de resync manuel.

## Acceptance Criteria

- [ ] Input `type="url"` pour `repo_url` dans `LumenView.vue` meta row :
  - Placeholder : `https://github.com/owner/repo`
  - Label : "Projet d'exemple"
  - Validation : format URL HTTPS github.com au blur
  - Binding v-model au state editeur + dirty tracking
- [ ] Badge d'etat visible apres un snapshot reussi :
  - "Snapshot : X fichiers · Y Ko · il y a Z jours"
  - Lien vers le repo dans un nouvel onglet
  - SHA court du commit snapshote
- [ ] Bouton "Resynchroniser" (icone `RefreshCw` lucide) a cote du badge :
  - Uniquement visible si un snapshot existe
  - Au clic : appelle `refreshSnapshot`, affiche un spinner, toast succes ou erreur
  - Message de confirmation si un snapshot different est detecte (SHA change)
- [ ] Le champ URL est `disabled` pendant un snapshot en cours
- [ ] Erreurs affichees via le systeme de toast existant avec messages actionables (ex. "Repo introuvable : verifie que le repo est public")
- [ ] Au blur de l'input avec une nouvelle URL valide : l'URL est sauvegardee via `setCourseRepo` (autosave comme le reste des champs)

## Technical Details

- Reutiliser le pattern de la meta row v2.22.0 (lignes ~795-930 de `LumenView.vue`)
- Le champ est visible uniquement en mode editor (pas dans la sidebar ni le reader)
- Le badge et le bouton resync sont extraits en mini-composant si ca depasse 50 lignes de template
- Gestion du loading state via un `ref<boolean>` local au composant parent
- Integration dans le flow de publication : si `repo_url` est defini mais pas de snapshot, la publication declenche automatiquement le snapshot (cf. backend tache 126)

## Dependencies

- Tache 127 (IPC + store actions)
