# Task: Migration DB + model lumen_courses (colonnes snapshot)

## Description

Ajouter les colonnes necessaires au stockage du snapshot d'un repo git exemple sur la table `lumen_courses`, et etendre le model CRUD pour lire/ecrire ces champs.

## Acceptance Criteria

- [ ] 5 colonnes ajoutees a `lumen_courses` via migration idempotente :
  - `repo_url TEXT NULL`
  - `repo_snapshot TEXT NULL` (JSON serialise)
  - `repo_commit_sha TEXT NULL`
  - `repo_default_branch TEXT NULL`
  - `repo_snapshot_at TEXT NULL` (ISO datetime)
- [ ] Migration retrocompatible : les cours existants restent intacts, les colonnes sont NULL par defaut
- [ ] Model `server/db/models/lumen.js` expose `updateRepo(courseId, { url, snapshot, sha, branch, at })` et lit les 5 champs dans `getCourse`/`listCourses`
- [ ] Tests unitaires du model couvrant create/read/update des nouveaux champs

## Technical Details

- Pattern existant : voir les migrations dans `server/db/schema.js` (chaque migration est une function idempotente avec check via `PRAGMA table_info`)
- `repo_snapshot` est un TEXT pour contenir le JSON serialise (< 10 Mo apres base64)
- Le `JSON.parse` du snapshot se fait au niveau du model, pas au niveau SQL
- Ne pas forcer d'index : aucune requete ne cherchera sur ces colonnes
