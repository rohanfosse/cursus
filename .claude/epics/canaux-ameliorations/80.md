# Task: Backend archivage canaux

## Description

Ajouter le support de l'archivage des canaux cote backend. Cela comprend :

- Migration pour ajouter la colonne `archived` (INTEGER DEFAULT 0) sur la table `channels`
- Fonctions `archiveChannel(id)`, `restoreChannel(id)`, `getArchivedChannels(promoId)` dans le modele
- Filtrer `getChannels()` avec `WHERE archived = 0` pour ne pas retourner les canaux archives par defaut
- Endpoint `POST /channels/:id/archive` (toggle archive/restore)
- Bloquer l'envoi de messages dans un canal archive (retourner une erreur 403)

## Acceptance Criteria

- [ ] La colonne `archived` existe sur la table `channels` avec valeur par defaut 0
- [ ] `archiveChannel(id)` met `archived = 1` et retourne le canal mis a jour
- [ ] `restoreChannel(id)` met `archived = 0` et retourne le canal mis a jour
- [ ] `getArchivedChannels(promoId)` retourne uniquement les canaux archives d'une promo
- [ ] `getChannels()` filtre les canaux archives (`WHERE archived = 0`)
- [ ] L'endpoint `POST /channels/:id/archive` fonctionne pour archiver et restaurer
- [ ] L'envoi de message dans un canal archive retourne une erreur 403

## Technical Details

- Modifier : `server/db/schema.js` (migration colonne `archived`)
- Modifier : `server/db/models/promotions.js` (fonctions archive/restore/getArchived, filtrer getChannels)
- Modifier : `server/routes/promotions.js` (endpoint archive)
- Modifier : handler d'envoi de message pour verifier `archived` avant insertion
