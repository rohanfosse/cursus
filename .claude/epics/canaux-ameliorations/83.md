# Task: Frontend archivage sidebar

## Description

Implementer le frontend de l'archivage des canaux dans la sidebar :

- Ajouter "Archiver" / "Restaurer" dans le menu contextuel des canaux
- Filtrer les canaux archives dans la sidebar (ne pas les afficher par defaut)
- Creer une section "Canaux archives" visible uniquement pour les responsables
- Rendre un canal archive en lecture seule (`isReadonly`) : desactiver la saisie de messages

## Acceptance Criteria

- [ ] Le menu contextuel d'un canal affiche "Archiver" ou "Restaurer" selon l'etat
- [ ] Les canaux archives ne sont pas affiches dans la liste principale de la sidebar
- [ ] Une section "Canaux archives" est visible pour les responsables
- [ ] Un canal archive est en lecture seule (la zone de saisie est desactivee)
- [ ] Le flag `isReadonly` est correctement propage a MessagesView

## Technical Details

- Modifier : `Sidebar.vue` (menu contextuel, filtrage, section archives)
- Modifier : `useSidebarActions.ts` (action archiver/restaurer via endpoint backend)
- Modifier : `MessagesView.vue` (mode lecture seule si canal archive)
- Depend du backend archivage (001) pour l'endpoint et le filtrage
