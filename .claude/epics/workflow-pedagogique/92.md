# Task: Etendre scheduler pour publier les devoirs

## Description

Etendre `server/services/scheduler.js` pour verifier toutes les 30s les devoirs avec `scheduled_publish_at <= now() AND published = 0`. A la publication : mettre `published = 1`, emettre socket event `devoir:published`, et optionnellement envoyer un message d'annonce dans le canal lie.

## Acceptance Criteria
- [ ] Le scheduler verifie les devoirs a publier a chaque tick (30s)
- [ ] Un devoir programme est publie automatiquement a l'heure prevue (+-1 min)
- [ ] Socket event `devoir:published` emis vers la room `promo:{promoId}`
- [ ] Message d'annonce envoye dans le canal du devoir (si channel_id existe)
- [ ] Le champ `scheduled_publish_at` est mis a null apres publication
- [ ] Pas de double-publication (guard `published = 0`)
- [ ] Logs serveur pour chaque publication programmee

## Technical Details

- Fichier : `server/services/scheduler.js`
- Pattern existant : `checkScheduledMessages()` -- creer `checkScheduledDevoirs()` similaire
- SQL : `SELECT * FROM travaux WHERE scheduled_publish_at <= datetime('now') AND published = 0`
- Update : `UPDATE travaux SET published = 1, scheduled_publish_at = NULL WHERE id = ?`

- Tache 1 (colonne scheduled_publish_at)
